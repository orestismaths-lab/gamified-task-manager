# 🖥️ Local Network Deploy – Task Manager στο δικό σου μηχάνημα

Στόχος: η εφαρμογή να τρέχει **στο PC σου** και να μπορείς να δίνεις ένα URL σε συναδέλφους στο ίδιο εταιρικό δίκτυο, ώστε:

- όλοι να βλέπουν το ίδιο σύστημα tasks
- να κάνουν **assign ο ένας στον άλλον** tasks

Δεν χρειάζεται cloud, μόνο το δικό σου μηχάνημα + εταιρικό δίκτυο.

---

## 1. Τι χρειάζεσαι

- Windows PC (αυτό που ήδη χρησιμοποιείς)
- Node.js 18+
- Δικαιώματα να “ακούς” σε port 3000 (συνήθως OK)

Η βάση δεδομένων είναι ένα SQLite file (`prisma/dev.db`) μέσα στο project – δεν χρειάζεται επιπλέον setup.

---

## 2. Εκκίνηση σε "LAN mode" (ώστε να βλέπουν και άλλοι)

### A. Development (για δοκιμές)

1. Άνοιξε `cmd` ή PowerShell στον φάκελο `task_manager`
2. Τρέξε:

```bash
npm install
npm run dev:lan
```

Αυτό ξεκινά Next.js dev server στο `0.0.0.0:3000`, που σημαίνει:

- Στο δικό σου PC: `http://localhost:3000`
- Από άλλο PC στο ίδιο δίκτυο: `http://ΟΝΟΜΑ_ΥΠΟΛΟΓΙΣΤΗ_ΣΟΥ:3000` ή `http://IP_ΤΟΥ_PC_ΣΟΥ:3000`

> Σημείωση: Αν σε κόβει firewall, δες παρακάτω το section **Firewall**.

### B. Production mode (για πιο “σοβαρή” χρήση)

1. Διπλό κλικ στο `START_PROD.bat` (μέσα στο `task_manager`)
2. Το script θα:
   - ελέγξει Node.js
   - τρέξει `npm run build`
   - ξεκινήσει τον server με `npm run start:lan`
3. Στην κονσόλα θα δεις οδηγίες τύπου:

```text
Local:   http://localhost:3000
Network: http://YOUR_PC_NAME:3000  (ή http://YOUR_IP:3000)
```

Αυτό είναι το URL που δίνεις στους άλλους χρήστες.

---

## 3. Firewall – αν οι άλλοι δεν μπορούν να μπουν

Αν στο δικό σου PC ανοίγει η εφαρμογή, αλλά οι άλλοι στο δίκτυο δεν μπορούν:

1. Πήγαινε **Windows Defender Firewall → Allow an app through firewall**
2. Βεβαιώσου ότι:
   - `node.exe` ή `Node.js` επιτρέπεται σε **Private** (και αν χρειάζεται σε **Domain**)
3. Αν σε ρωτήσει firewall την πρώτη φορά που τρέχεις `npm run dev:lan` ή `START_PROD.bat`, διάλεξε **Allow** για Private/Domain δίκτυο.

Αν υπάρχει εταιρικό firewall/proxy, ίσως χρειαστεί βοήθεια από IT για να ανοίξουν το port 3000 από άλλους clients προς το PC σου.

---

## 4. Multi-user & Assignments – πώς λειτουργεί

- Κάθε χρήστης κάνει **Register** με το δικό του email/password.
- Στο σύστημα δημιουργούνται **μέλη (members)** – μπορεί να είναι άτομα ή ρόλοι (π.χ. “Maria”, “Team A”).
- Όταν δημιουργείς task, μπορείς να:
  - επιλέξεις **owner/assignee** από τη λίστα μελών
  - να αλλάξεις assigned member αργότερα (edit task)
- Όλοι οι χρήστες που μπαίνουν στο ίδιο URL δουλεύουν πάνω στο **ίδιο σύνολο tasks/members** (κοινή DB στο PC σου).

> Άρα, για να κάνετε assign ο ένας στον άλλον, απλώς μοιράζεστε το ίδιο URL του server σου.

---

## 5. Τι δίνω στους άλλους χρήστες

Στείλε τους π.χ. με email/Teams:

> “Για το Task Manager, μπείτε στο:  
> `http://ΤΟ-ΟΝΟΜΑ-ΤΟΥ-PC-ΜΟΥ:3000`  
> (δουλεύει μόνο μέσα στο εταιρικό δίκτυο)"

Προαιρετικά, μπορείς να δημιουργήσεις ένα μικρό `README_FOR_USERS.md` με:

- URL της εφαρμογής
- Πώς κάνουν Register/Login
- 2‑3 bullets για δημιουργία/assign tasks

---

## 6. Πώς το σταματάω / ξαναξεκινάω

- Για να σταματήσεις τον server: πήγαινε στο παράθυρο της κονσόλας και πάτα **Ctrl + C**.
- Για να τον ξαναξεκινήσεις:
  - είτε τρέχεις πάλι `npm run dev:lan`
  - είτε διπλό κλικ στο `START_PROD.bat`

---

Με αυτά τα βήματα, η εφαρμογή είναι ουσιαστικά “productized” για usage μέσα στο δικό σου δίκτυο, χωρίς εξωτερικά services. 🎯
